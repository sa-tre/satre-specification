# La Arquitectura SATRE

La versión actual de la arquitectura puede consultarse [aquí](https://satre-archimate.readthedocs.io/en/latest/?view=id-4349bc52159b48e9b785e9809a876c03).

Esta arquitectura estándar para entornos de investigación confiables (TRE) proporciona una arquitectura integral de alto nivel para organizaciones de investigación que gestionan datos sensibles de forma segura.

La arquitectura está documentada utilizando el lenguaje de modelado [ArchiMate](https://pubs.opengroup.org/architecture/archimate3-doc/)con modelos creados mediante la herramienta de modelado de código abierto [Archi](https://www.archimatetool.com/).

Las capacidades que una organización requiere para operar un TRE se documentan y se descomponen para mostrar los elementos necesarios para materializar dichas capacidades (roles, procesos, aplicaciones y datos).

La arquitectura y el estándar SATRE se corresponden con el metamodelo de arquitectura que se muestra a continuación.

Se proporcionan vistas alineadas con las capacidades, y vistas adicionales que muestran la alineación con marcos externos, como el [Five Safes Framework](https://ukdataservice.ac.uk/help/secure-lab/what-is-the-five-safes-framework/).

Los archivos de ArchiMate se gestionan en el [Repositorio de arquitectura](https://github.com/sa-tre/satre-archimate).

Un informe sobre la arquitectura está disponible en [zenodo](https://zenodo.org/records/10053383).
