# Alineación de controles de SATRE

La [tabla de alineación de controles centrada en SATRE para Entornos de Investigación Confiables (TRE)](https://zenodo.org/records/16837077)respalda el cumplimiento de SATRE de forma eficiente y transparente. Proporciona vínculos entre los requisitos de SATRE y los controles de otros marcos relevantes: ISO 27001, Data Security and Protection Toolkit (DSPT) y Digital Economy Act (DEA).

## Por qué es importante la alineación de controles de SATRE

La operación de un TRE implica desenvolverse en múltiples marcos, normativas y documentos de orientación. La referencia de alineación de controles le ayuda a:

- Identificar dónde los controles de los TRE se superponen entre distintos marcos, y qué controles de SATRE están alineados con controles existentes de ISO, DSPT, SDE (_Secure Data Environment_) y DEA.
- Comprender qué áreas ya están cubiertas por otros estándares.
- Detectar brechas específicas con respecto a SATRE.

Esta referencia debe actuar como una herramienta de orientación, que ayude a los operadores y desarrolladores de TRE a identificar rápidamente en qué puntos las buenas prácticas de otros marcos contribuyen a las capacidades de SATRE. Puede entenderse como un mapa para el cumplimiento de SATRE: destaca las conexiones entre estándares, muestra dónde las prácticas existentes ya están contribuyendo y señala dónde se requiere trabajo adicional, sin partir de supuestos sobre el cumplimiento de otros marcos.


:::{important}
Esta referencia se proporciona únicamente con fines orientativos. Sirve como punto de referencia para la alineación de controles de SATRE y no implica cumplimiento de ISO 27001, del DSPT ni del DEA. Cada marco requiere una evaluación independiente.
:::
